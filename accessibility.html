Accessibility Statement

Arnon is committed to making its website and application accessible to all users, including people with disabilities, in accordance with WCAG 2.1 Level AA guidelines.

What we've done

  • ARIA labels on all interactive elements (buttons, inputs, dialogs)
  • Keyboard navigation support with visible focus indicators
  • Screen reader compatibility — live regions for new messages and alerts
  • Semantic HTML — proper table headers, roles, and landmarks
  • Responsive design — works on all screen sizes
  • No user-scalable restrictions — users can zoom freely
  • Sufficient color contrast for text and interactive elements
  • Light and dark mode support on the landing page via system preference

Known limitations

  • Voice notes require microphone access and may not be available on all assistive devices
  • Voice notes may not work in Tor Browser
  • Audio playback controls use the browser's native player

Feedback

If you encounter any accessibility issues, please open an issue on GitHub.
